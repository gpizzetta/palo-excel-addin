Palo OLAP Add-in

Ce prototype implemente la V1 du cahier des charges: connexions Palo, test de connexion et fonctions Excel PALO_*.

Point d'entree UI: taskpane.html.

Menu contextuel « Palo action »

La commande est declaree dans le manifeste (ExtensionPoint ContextMenu + OfficeMenu id="ContextMenuCell") et executee par commands.js (Office.actions.associate("paloRibbonAction", …)), comme le bouton du ruban.

Excel bureau (Windows / Mac) : Microsoft utilise un menu contextuel different pour les cellules situees dans un Tableau Excel (insertion > Tableau). Dans ce cas, les entrees des complements peuvent ne pas apparaitre — limitation plateforme (office-js#6612). Essayez un clic droit sur une cellule hors tableau, ou utilisez le bouton Palo action dans l’onglet Accueil.

Apres modification du manifeste : recharger le complement (catalogue d’administration ou sideload) et vider le cache si besoin.

Canal BETA (staging)

Tests sans toucher la production : manifest staging — complement Palo OLAP Add-in (BETA) (Id Office different, peut coexister avec la prod).

Erreur « Sorry, something went wrong » a l'ouverture

Souvent lie au recalcul massif des formules PALO.* pendant le chargement du complement. Depuis la 1.0.2.8 : delai de grace a l'ouverture, pas de double chargement palo-api.js, page unique shared runtime.

  1. Retirer le complement Palo, redemarrer Excel, reinstaller le manifeste a jour (manifest.xml).
  2. Ouvrir le fichier sans complement (ou desactiver temporairement les complements) pour recuperer le contenu.
  3. Après ouverture : attendre quelques secondes puis Ctrl+Alt+F9 pour recalculer les formules Palo.
  4. Ouvrir le volet Connexion et selectionner une connexion active avant le recalcul.

Erreur « Sorry, something went wrong » (autres cas)

Message generique Excel (souvent au chargement du complement ou d'un classeur avec formules PALO.*). Pistes :

  1. Retirer le complement Palo, redemarrer Excel, reinstaller le manifeste a jour (manifest.xml).
  2. Ouvrir le fichier sans complement (ou desactiver temporairement les complements) pour recuperer le contenu.
  3. Utiliser Snapshot uniquement sur une copie : la version 1.0.2.7+ cree un fichier *_snapshot_*.xlsx separe sans modifier le classeur ouvert.

Le volet affiche la version des fichiers charges par Excel, pas celle de GitHub. Ctrl+F5 dans un navigateur ne vide pas le cache du WebView Excel.

  1. Verifier en ligne : version.json (version deployee) et manifest.xml (doit contenir Palo.Group.Snapshot).
  2. Dans le Centre d’administration Microsoft 365 : deployer le complement via l’URL du manifeste (pas seulement un fichier XML telecharge une fois). Mettre a jour / republier apres chaque release.
  3. Sur le poste : retirer le complement Palo, redemarrer Excel, le reinstaller.
  4. Ouvrir le volet Connexion : si le pied de page indique une version plus recente que le script embarque, le complement est encore en cache.
