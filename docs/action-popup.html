PALO.DATAC — écriture cube

Le mode 1 (default) est le splash par défaut du serveur : la valeur est répartie sur les bases sous le consolidé (logique Jedox / # en saisie manuelle — pas forcément parts égales). Les modes 2 et 3 correspondent à !! et ! : la valeur saisie est ajoutée (2) ou écrite telle quelle (3) sur chaque cellule de base — ce n’est pas une division du total entre enfants. Les modes 4 / 5 (!# / !!#) n’ont pas besoin d’un path de référence : ils ne touchent que les bases déjà peuplées ; si toutes les bases sont vides sur cette tranche, le serveur peut refuser (normal). Détail : Splashing overview, Variations # (!# !!#), docs/palo-like-copy-datac-action.md § 5.1.

Mode avancé — LIKE / COPY (chemins partiels → /cell/copy)

Formes reconnues : 300 like 2025 (seules les dimensions listées après like changent ; les autres reprennent le chemin de la cellule courante, comme l’add-in 2010) ou COPY Année:2025. Séparateur entre segments : ;. Ambiguïté : Dimension:élément. LIKE envoie GET /cell/copy avec value = le nombre (pas /cell/replace). La case use_rules du bloc « Copier depuis… » s’applique aussi ici. La formule PALO.DATAC n’est pas modifiée. Voir docs/palo-like-copy-datac-action.md § 5.2 et le cahier § G.5.
